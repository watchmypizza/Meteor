Install

Set up in minutes

Requirements

  • Python 3.13
  • Ollama + model (e.g. gemma3:4b)
  • Firebase service account JSON
pip install -r requirements.txt

AI Moderation with Ollama

  • If you do not have ollama installed, you can download it here:
    • Ollama Download

  • After installing Ollama, we have to install the model. We will use gemma3:4b for this
ollama serve & ollama pull gemma3:4b

    Please keep in mind, the next steps are optional and not required.

  • ollama serve has to keep running in the background so it is active. This is default for windows, on linux you can activate a service so it starts on system-startup.

  • We first of all need to create a user and usergroup for ollama
sudo useradd -r -s /bin/false -U -m -d /usr/share/ollama ollama sudo usermod -a -G ollama $(whoami)
  • After we have added the user and usergroup for ollama, we will create the service file
sudo touch /etc/systemd/system/ollama.service
  • We now have to write to the service file the following content
[Unit] Description=Ollama Service After=network-online.target [Service] ExecStart=/usr/bin/ollama serve User=ollama Group=ollama Restart=always RestartSec=3 Environment="PATH=$PATH" [Install] WantedBy=multi-user.target
  • We now have to start the service after creating it with the following command
sudo systemctl daemon-reload sudo systemctl enable ollama
  • You should now be all set to use the AI Moderation feature!

Environment

Create .env in project root:

TOKEN=YOUR_DISCORD_BOT_TOKEN FIREBASE_JSON=JSONS/your-service-account.json

  • /configure welcomer - welcome/goodbye messages
  • /configure announcement - set announcement channel for $ wall
  • /configure modlogs - dedicated moderation logs
  • /configure botrole - role to exclude from member counts
  • /configure suggestions - auto-react ‚úÖ/‚ùå
  • /configure verifiedrole - set verified role
